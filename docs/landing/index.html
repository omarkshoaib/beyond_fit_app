§ 01 How it works

From profile to plate, in five
deterministic steps.

01·step

Onboard

Pick an avatar — powerlifter, powerbuilder, general fitness. Choose 3 to 6 training days, your experience level, and any injuries. The engine reads each as a constraint, not a preference.

~ 60 sec
4 questions
02·step

Resolve the split

Day templates are picked from a fixed table indexed by avatar × frequency. Push / Pull / Legs for some; Upper / Lower / Push / Pull for others. The split is never randomised.

deterministic
tabular lookup
03·step

Budget the volume

Per muscle group, weekly sets are gated between MEV (minimum effective) and MRV (max recoverable), scaled by experience level. Deload weeks (every fifth) drop the budget by 40%.

MEV · MRV
by week
04·step

Fill the slots

Each day gets three tiers: a main compound (highest fatigue), a secondary, an isolation. Movement-pattern matching, biomechanical synergy, and a per-day fatigue cap keep the session honest.

3 tiers · 12 fatigue
per day
05·step

Auto-regulate

You log each set. Next week, the autoregulator reads your actual RPE against the target and corrects the load — minus 4% per RPE point overshot, plus the progressive overload increment. Math, not feel.

RPE error · 4%
per point
§ 02 The math, plainly

No black boxes. Read the source.

autoregulator.py · L 30

# Next-week load given last week's actual effort.
def calculate_next_load(
    last_weight, last_target_rpe,
    last_actual_rpe, next_target_rpe,
):
    err = last_actual_rpe - last_target_rpe
    base = last_weight * (1 - err * 0.04)
    jump = next_target_rpe - last_target_rpe
    nxt  = base + jump * 0.025 * base
    return round(nxt / 2.5) * 2.5
A single function. Same inputs always produce the same output. No hidden state.
